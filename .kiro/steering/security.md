# セキュリティ標準

本プラットフォームは個人情報・評価情報・報酬情報を扱うため、
通常のWebアプリより一段高いセキュリティ水準を要求する。

## セキュリティ哲学
- **Secure by Default** — デフォルトで拒否、明示的に許可
- **最小権限の原則** — ロールが必要最小限の権限のみを持つ
- **深層防御** — middleware・サービス層・DB層の各層で検証
- **評価の匿名性保護** — 被評価者APIには評価者の識別情報を絶対に含めない

## 認証

### パスワードポリシー
- 最低12文字以上
- 英大文字・英小文字・数字・記号から3種以上を含む
- 過去5世代のパスワードと一致するものは拒否
- bcrypt（コストファクター12以上）でハッシュ化

### セッション
- セッション有効期限: 8時間
- アイドルタイムアウト: 30分
- ログアウト時はセッショントークンを即時失効
- 異なるデバイスでの同時ログインを許可（セッション一覧から手動失効可）

### 将来拡張
- 2FA（TOTP）: MUST（エンタープライズ導入の条件）
- SSO（SAML/OIDC）: SHOULD（組織規模に応じて有効化）

## 認可（RBAC）

### ロール階層
```
ADMIN > HR_MANAGER > MANAGER > EMPLOYEE
```

### 権限マトリクス（抜粋）
| 操作 | ADMIN | HR_MANAGER | MANAGER | EMPLOYEE |
|------|-------|-----------|---------|----------|
| マスタ管理 | ✅ | ❌ | ❌ | ❌ |
| 組織編集 | ✅ | ✅ | ❌ | ❌ |
| 全社員データ閲覧 | ✅ | ✅ | ❌ | ❌ |
| 部下データ閲覧 | ✅ | ✅ | ✅（部下のみ） | ❌ |
| 自己データ閲覧 | ✅ | ✅ | ✅ | ✅ |
| 評価生データ閲覧 | ✅ | ✅ | ❌ | ❌ |
| 総合評価修正（確定後） | ✅ | ❌ | ❌ | ❌ |

### 検証箇所（深層防御）
1. **middleware層**: URL/ルート単位でロール判定（早期拒否）
2. **サービス層**: リソースレベルの所有権判定（「この部下は自分の直属か」）
3. **DB層**: Prisma の WHERE 句に常に `ownerId` / `userId` を含める（所有権制限）

### マルチテナンシー方針
- **単一テナント前提**（1社に1インスタンス）
- `tenantId` は導入しない
- 将来マルチテナント化する場合は全テーブルに `tenantId` を追加し、PostgreSQL RLS で制御

## 入力バリデーション

- 全外部入力を Zod スキーマで検証
- サーバーサイドバリデーション必須（クライアント側は補助のみ）
- ホワイトリスト方式（許可する値・形式を定義）
- SQLインジェクション防止: Prisma のパラメータバインディングのみ使用（生SQLは禁止）
- XSS防止: React のデフォルトエスケープを信頼、`dangerouslySetInnerHTML` は原則禁止
- ファイルアップロード: MIMEタイプ・ファイルサイズ・拡張子のホワイトリスト検証

## 機密データ保護

### PII（個人識別情報）
- 氏名・社員番号・メール・評価コメントはすべて PII 扱い
- DBカラムレベルで AES-256 暗号化
- ログへの PII 出力禁止（ログ出力前にマスキング）
- AI API 送信時は氏名・社員番号を匿名トークン（例：`emp_001`）に置換

### シークレット管理
- 平文でのシークレット保存禁止
- 環境変数またはシークレットマネージャー（AWS Secrets Manager / Vercel Env）使用
- `.env.local` は git 管理外（`.gitignore` 必須）

### 評価者の匿名性
- DB: `evaluation_responses.evaluator_id` は保持（HR_MANAGER以上のみ参照可）
- API: 被評価者向けレスポンスには `evaluator_id` を一切含めない
- マイルド化変換後のコメントからも評価者特定可能な表現を除去

## 通信セキュリティ
- HTTPS 必須（HSTS ヘッダーで強制）
- CORS: 許可オリジンを環境変数で明示定義
- セキュリティヘッダー: CSP, HSTS, X-Frame-Options: DENY, X-Content-Type-Options: nosniff

## レート制限
- 全APIに適用: 1ユーザーあたり毎分100リクエスト
- ログインAPI: 1IPあたり毎分10回（ブルートフォース対策）
- AI APIを呼び出すエンドポイント: 1ユーザーあたり毎分20回（コスト保護）

## 監査ログ

### 記録対象
- 認証イベント（ログイン成功/失敗、ロックアウト）
- 権限昇格・ロール変更
- マスタデータの変更（スキル・役職・等級・ウェイト）
- 評価データの確定・修正
- 個人情報エクスポート

### 記録内容
- 誰が（user_id）
- 何を（action + resource_type + resource_id）
- いつ（timestamp）
- どこから（IP + User-Agent）
- 変更前後の値（差分）

### 保持期間
- 監査ログ: 7年間保持（人事情報の法定保持期間準拠）
- アクセスログ: 1年間保持

## データ保持・削除
- 在職社員データ: 無期限保持
- 退職社員データ: 在籍時データを7年間保持後、匿名化処理
- 本人からの削除要求: GDPR/個人情報保護法に準拠した削除フローを提供

## インシデント対応
- セキュリティイベントを監視システムに送信（認証失敗・権限エラー・レート制限到達）
- 閾値超過時にADMINへアラート通知
- インシデント発生時は影響範囲の特定 → 該当セッション失効 → ユーザー通知の順で対応

# AI 連携標準

AI を評価品質の担保・心理的安全性の確保に活用するため、
利用方針・プロンプト設計・個人情報保護・コスト制御を統一ルール化する。

## 哲学
- **AI はツール、決定するのは人間** — 評価の最終確定は必ず人間が行う
- **透明性** — AI がどこで・何のために使われているかをユーザーに明示
- **プライバシーファースト** — 個人識別情報は AI に送信しない
- **モデル中立** — 特定 AI プロバイダにロックインしない設計

## AI 利用箇所

| 機能 | ドメイン | 目的 | モデル種別 |
|------|---------|------|-----------|
| 対話型品質ゲート | `ai-coach` | 評価コメントの具体性・価値観関連性の判定と改善提案 | Chat（低〜中レベル） |
| マイルド化変換 | `feedback` | ネガティブ表現を建設的な表現に書き換え | Chat（中レベル） |
| FB要約生成 | `feedback` | 評価コメント群から強み・改善点のサマリー生成 | Chat（中レベル） |

**それ以外のドメインからの AI 呼び出しは禁止**。必要になった場合は設計レビューを経て `ai-coach` または `feedback` に集約する。

## プロンプト設計原則

### 個人情報除去
AI API に送信するデータから以下を必ず除外する:
- 氏名（匿名トークンに置換: `emp_001`）
- 社員番号
- メールアドレス
- 電話番号
- 住所

### プロンプトの構造
```
[システム指示]
- 役割定義
- タスク定義
- 出力形式の制約

[コンテキスト]
- 評価対象の役職・等級（匿名）
- 評価項目の定義

[入力]
- 評価者が書いたコメント（匿名化済み）

[出力形式]
- JSON形式を強制（例: {"quality_ok": true, "suggestions": [...]}）
```

### 出力の検証
- JSONパース失敗時はリトライ（最大2回）
- 期待するスキーマから外れた出力は拒否してフォールバック
- 禁止トークン（差別用語・機密情報推測）を含む場合は出力を破棄

## モデル切替設計

```
lib/ai-coach/providers/
  ├── anthropic.ts   # Claude API
  ├── openai.ts      # OpenAI API
  └── index.ts       # AI_PROVIDER 環境変数で切替
```

- プロバイダ層は共通インターフェース（例: `ChatProvider`）で抽象化
- 呼び出し元は `AI_PROVIDER=claude` などの設定のみで切替可能
- モデル変更時の回帰は評価ログ（対話ターン・品質判定結果）でE2Eテスト

## パフォーマンス

### タイムアウト
- 対話型品質ゲート: 5秒（超過時は手動モードに退避）
- マイルド化変換: 10秒（非同期ジョブのため若干緩い）
- FB要約生成: 15秒

### 非同期処理
- マイルド化変換・要約生成は BullMQ ジョブキューで非同期実行
- ユーザー待機が必要な対話型品質ゲートのみ同期呼び出し

## コスト管理

### トークン使用量の記録
- 全 AI 呼び出しに対して以下をログ記録:
  - user_id, domain, prompt_tokens, completion_tokens, total_cost_usd
- 月次・週次・ユーザー別の集計ダッシュボード（管理者向け）

### アラート閾値
- 月次コスト: 予算の80%到達でWARN、100%到達でCRITICAL
- ユーザー別異常値: 通常の5倍以上で調査フラグ

### コスト最適化
- 同一入力に対する結果をキャッシュ（Redis）
- 長いコメントは要約前処理してからマイルド化変換
- プロンプトキャッシュ（Anthropic）を活用してシステムプロンプトのコストを削減

## テスト戦略

- **ユニットテスト**: プロンプト構築ロジック・匿名化処理・パーサー
- **統合テスト**: モック AI クライアントでフロー検証
- **スナップショットテスト**: 代表的な入力に対する AI 出力の回帰検知
- **E2Eテスト**: 実 API を使った主要シナリオ（ステージング環境のみ、コスト制約あり）

## 禁止事項

- AI に個人識別情報（氏名・社員番号等）をそのまま送信すること
- AI の出力をそのまま確定データとして保存すること（人間レビューを必ず介在）
- 評価の最終スコアを AI に算出させること
- ユーザープロンプトに直接 AI API キーを含めること
- AI レスポンスをエスケープせずに DOM に挿入すること
